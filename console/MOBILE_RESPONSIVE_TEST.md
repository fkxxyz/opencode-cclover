# 双栏消息布局 + 移动端适配测试指南

## 实现内容

已完成前端双栏布局实现，支持查看员工与不同对象的对话，并针对移动端做了响应式适配。

### 新增组件

1. **ConversationList** (`console/src/components/employee/ConversationList.tsx`)
   - 显示对话对象列表
   - 支持选择对话
   - 实时更新对话列表
   - ✨ 响应式：移动端占满宽度，桌面端固定 280px

2. **MessagePanel** (`console/src/components/employee/MessagePanel.tsx`)
   - 显示与选定对象的消息
   - 消息气泡样式
   - 实时接收新消息
   - ✨ 移动端：显示返回按钮（左上角 `<` 图标）

3. **ConversationView** (`console/src/components/employee/ConversationView.tsx`)
   - 组合 ConversationList 和 MessagePanel
   - 管理选中状态
   - ✨ 响应式布局切换：
     - 桌面端（≥ md）：双栏并排显示
     - 移动端（< md）：单栏切换显示

### 新增 Hook

- **usePeers** (`console/src/hooks/usePeers.ts`)
  - 获取对话对象列表
  - 实时更新（收到新消息时自动添加新对话对象）

### 修改内容

- **EmployeeDetail** 页面：使用 `ConversationView` 替换原有的 `MessageList`
- **types/index.ts**：添加 `Peer` 类型定义

## 响应式设计

### 断点定义

使用 MUI 默认断点：
- **xs**: 0px - 600px（手机竖屏）
- **sm**: 600px - 900px（手机横屏/小平板）
- **md**: 900px - 1200px（平板）
- **lg**: 1200px - 1536px（桌面）
- **xl**: 1536px+（大屏）

本实现使用 `md` (900px) 作为分界点。

### 桌面端布局（≥ 900px）

```
┌─────────────────────────────────────────────────────────────┐
│  消息通信                                                     │
├──────────────┬──────────────────────────────────────────────┤
│ 对话列表      │  与 bayecao 的对话                            │
│ (280px固定)  │  (flex: 1)                                    │
│              │                                               │
│ ☑ bayecao    │  [消息气泡]                                   │
│ ☐ alice      │                                               │
│ ☐ bob        │                                               │
│              │                                               │
└──────────────┴──────────────────────────────────────────────┘
```

## 技术细节

### 响应式实现

使用 MUI 的 `useMediaQuery` Hook：

```typescript
const theme = useTheme()
const isMobile = useMediaQuery(theme.breakpoints.down("md"))
```

### 条件渲染

```typescript
{isMobile ? (
  // 移动端：单栏切换
  <>{!selectedPeer ? <ConversationList /> : <MessagePanel />}</>
) : (
  // 桌面端：双栏并排
  <><ConversationList /><MessagePanel /></>
)}
```

### 返回按钮

仅在移动端且有 `onBack` 回调时显示：

```typescript
{isMobile && onBack && (
  <IconButton onClick={onBack}>
    <ArrowLeft />
  </IconButton>
)}
```
